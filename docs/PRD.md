# 產品需求文件 (PRD) - 食譜收藏夾系統

## 1. 專案概述
- **背景與動機**：在煮飯時經常會需要參考過去做過的料理或在網路上看到的食譜，但散落各處難以集中管理。建立一個專屬的食譜收藏系統可以方便快速查找並記錄食譜。
- **目標用戶**：我（個人使用者）。
- **核心價值主張**：提供一個輕量、直覺的食譜集中地，能快速紀錄與查詢食譜及所需食材，提升備餐效率。

## 2. 功能需求

1. **查詢食譜**
   - 作為 **使用者**，我希望 **能搜尋或瀏覽食譜**，以便 **在煮飯時快速找到想做的料理**。
2. **新增食譜**
   - 作為 **使用者**，我希望 **能新增食譜（包含名稱、步驟、所需食材等）**，以便 **收藏自己喜歡的料理配方**。
3. **列出食材**
   - 作為 **使用者**，我希望 **能檢視食譜中的食材清單**，以便 **在外出採買時確認要買什麼材料**。
4. **編輯與刪除食譜**
   - 作為 **使用者**，我希望 **能修改或刪除現有的食譜**，以便 **隨時更新更好的做法或移除不常做的菜色**。
5. **食譜分類與標籤**
   - 作為 **使用者**，我希望 **能幫食譜加上分類或標籤（如：主菜、湯品、備餐）**，以便 **更有條理地整理與篩選食譜**。

## 3. 非功能需求
- **技術限制**：預計使用 Python (Flask) + Jinja2 + SQLite 作為基礎架構開發。
- **效能與安全考量**：
  - 系統需確保回應速度快，介面簡潔無負擔。
  - 執行危險操作（如刪除食譜）時需有防呆確認機制，避免誤刪資料。

## 4. MVP 範圍 (Minimum Viable Product)
- **Must Have (必須有)**：
  - 新增食譜（名稱、做法、食材）。
  - 檢視所有食譜清單。
  - 檢視單一食譜的詳細資訊（含列出食材）。
- **Should Have (應該有)**：
  - 編輯與刪除現有食譜。
  - 基本的關鍵字搜尋功能。
- **Nice to Have (加分項)**：
  - 食譜的分類或標籤篩選。
  - 依據多個食譜自動合併生成「總採買食材清單」。
